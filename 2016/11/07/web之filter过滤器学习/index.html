web之filter过滤器学习

2016-11-07 · 92 words · 1 minute read

> 定义：过滤器是一个服务器的组件，它可以截取用户端的请求与响应信息，并对这些信息过滤。

> Servlet过滤器是在 java Servlet 2.3 中定义的，它能够对Servlet容器中的请求和响应对象，进行检查和修改

> Servlet过滤器并不会生成request和response对象，它只起到过滤的作用

> Servlet过滤器会在请求还未到Servlet之前检查request对象,修改 Request Header 和 Request 内容；在Servlet被调用后检查Response对象，修改 Response Header 和 Response 内容

> Servlet过滤器可以过滤 Servlet , JSP , HTML 等web组件（JSP最终也是转换为Servlet的）

定义一个Servlet过滤器（Fileter）

写一个java类，实现 java.servlet.Filter接口 
实现三个方法：

    init(FilterConfig filterConfig) - - -初始化  

    doFilter(ServletRequest request,ServletResponse response,FilterChain chain)   - - - 完成实际的过滤操作  这个方法得到调用后，里面重要的一点是要调用chain.doFilter(request,response)方法，它可以将请求传递给下一个过滤器（或目标资源），或者转发、重定向到其他资源。                                                                                   
    destroy()  - - -过滤器销毁阶段  Servlet容器在销毁过滤器实例之前该方法得到调用，以释放占用的资源

过滤器工作过程：

web容器启动的时候会加载过滤器。过滤器启动。 当用户发送请求服务器时，并不会直接到服务器，而是先到过滤器，过滤器会进行判断是否符合请求规则。 如果符合规则，过滤器会将用户请求发送至web资源。 web资源对用户的请求进行响应时，也会先将资源响应发送至过滤器，过滤器再将web资源的响应发送给用户。

过滤器的生命周期

实例化：通过web.xml配置，容器启动会加载web资源，包括过滤器。过滤器被加载即完成实例化。只有当web容器启动时，过滤器才会进行实例化，即过滤器只会实例化一次。 初始化：init()–web容器创建过滤器实例后（过滤器实例化成功）会调用过滤器的init() 方法。这个方法中可以读取web.xml文件中过滤器的参数，进行初始化操作。同样只初始化一次。 过滤：doFilter()–过滤方法（过滤器的核心方法）。每次请求与过滤关联的url时，都会执行过滤方法。 销毁：destroy()–当web容器关闭时，在销毁过滤器前调用该方法。可以释放过滤器占用的资源。

    web.xml配置信息
    <filter>
        <filter-name>filter的名字</filter-name>
        <filter-class>filter的完全限定名</filter-class>
        <!-- 初始化参数 可以不写或是写1对或多对-->
        <init-param>
            <description>描述信息</description>
            <param-name>参数名称</param-name>
            <param-value>参数的值</param-value>
        </init-param>
    </filter>
    <filter-mapping>
        <filter-name>filter的名字</filter-name>
        <!-- 当用户请求的URL和指定的URL匹配时触发过滤器工作-->
        <url-pattern>URL</url-pattern>
        <!-- 可以不写或是写1对或多对 
            值为：REQUEST|INCLUDE|FORWARD|ERROR 默认REQUEST-->
        <dispatcher></dispatcher>
    </filter-mapping>
