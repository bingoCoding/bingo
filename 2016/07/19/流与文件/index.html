流与文件

2016-07-19 · 104 words · 1 minute read

流

输入流：在java api 中可以读入一个字节序列的对象叫输入流。 输出流：可以向其中写入一个字节序列的对象叫输出流。 这些可操作的字节序列可以是文件，网络连接，内存块 InputStream 和 outputStream 构成了有层次结构的输入/输出类的基础。 InputStream 有一个抽象方法 int read(); outputStream 有一个抽象方法 void write（）; read 和 writer都是阻塞式执行。意味着如果不能立即执行，那么当前的线程将被阻塞，这使得在这个方法等待的时间里，其他线程将有机会去执行有用的工作。 available方法可以检查当前可用于读入的字节数量。 int bytesAvailable =in.available(); byte[] data=new byte[bytesAvailable]; if(bytesAvailable>0){ in.read(); } 字节流不便于处理unicode存储的信息，所以从抽象类Reader和writer继承出了专门类 当完成流的读写时，应该调用close 来关闭流，释放资源

    一般情况下，程序员很少用到read,write,因为大部分数据可能更多的是数字、字符串、对象，而不是源生字节

    InputStream Api:
    int read()  从数据中读入一个字节，并返回该字节，碰到结尾返回-1。
    int read(byte[] b) 读入一个数组，并返回实际读入的字节数。最多读入b.length字节
    int read(byte[] b,int off,int len) off 偏移量，len读取长度
    long skip(long n) 在输入流中跳过n个字节，返回实际跳过的字节数
    int available() 返回不阻塞情况下的字节数
    void mark(int readlimit) 在输入流的当前位置打一个标记（并非所有流都支持），如果输入流中已经读入的字节多于readlimit个，则这个流允许忽略这个标记。
    void reset() 返回到最后的标记，随后对read的调用将重新读入这些字节。如果没有标记，则这个流不被重置。
    boolean markSupported() 如果这个流支持打标记，则返回true
    outputStream api:
    void write(int n) 写出一个字节的数据
    void write(byte[] b)
    void write(byte[] b,int off,int len) 写出字节到数组 b 中
    void close() 关闭流、
    void flush() 清空输出流，即将缓冲数据发到目的地

以上可以处理字节和字节数组，但当需要读写字符串和数字时就需要其他更强大的子类了。
如 dataInputStream ,DataOutputStream可以以二进制格式读取所有的基本java类。
如 ZipInputStream ,ZipOutputStream 可以以常见的zip压缩格式读写文件。
